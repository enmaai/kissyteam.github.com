Module¶

  • io

Method¶

io.post()¶
XHR post ( url , [ data , callback , dataType ] )
发送 http post 请求
Parameters:
 
  • url (string) – 请求地址
  • data (Object|string) – 请求附带的参数，参见 data .
  • callback (function) – 请求成功后的回调，参见 success .
  • dataType (string) – 传到回调函数作为参数的数据类型，参见 dataType
Returns:
 

代表本次请求的 xhrObj
Return type:
 

XhrObj
XHR upload post( url , [ callback , dataType ] )
data 可忽略，同上个函数描述.

实际上该函数是 io() 的 shortcut

io.post = function(url, data, callback, dataType) {
    // data 参数可省略
    if (S.isFunction(data)) {
        dataType = callback;
        callback = data;
        data = undefined;
    }

    return io({
        type:"post",
        url: url,
        data: data,
        success: callback,
        dataType: dataType
    });
};

Note

post 请求从来不会被缓存，因此 io.cfg.cache 配置可以不用配置.

Demo¶

请求页面 test.php , 但是忽略返回结果

KISSY.io.post("test.php");

请求页面 test.php , 并附带一些参数传递给后端

KISSY.io.post("test.php",{
    name:"john".
    time:"2pm"
});

alert 请求成功后返回的结果，数据类型由响应的 content-type 决定

KISSY.io.post("test.php",function(d){
    alert(d);
});

alert 请求成功后返回的 json 数据，和响应的 content-type 无关

KISSY.io.post("test.php",function(d){
    alert(d);
},"json");
